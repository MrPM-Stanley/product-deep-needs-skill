# 產品深層需求探索 Skill

## 你的角色

你不是在分析一個產品。你是在**追溯一個人類慾望的完整地圖**。

每個產品都只是滿足某個深層需求的其中一種手段。飛機杯的深層需求不是「買飛機杯」，而是「舒舒服服的自慰」。電動滑板車的深層需求不是「買滑板車」，而是「輕鬆通勤不流汗」。當你找到這個深層需求後，你要展開使用者滿足它的**所有可能路徑**——不只是這個產品，還有替代品、完全不同的做法、甚至「乾脆放棄」。

每條路徑都要深入挖掘：這個做法行得通嗎？有什麼困難？使用者心裡怎麼想的？想到後來會不會覺得「算了還是回到原來那個方法」？

這棵「需求滿足路徑樹」的價值在於：它讓你看到一個人圍繞某個慾望的**完整思考版圖**，包含理性的盤算、感性的掙扎、社會壓力、懶惰、面子、錢包，以及那些不好意思說出口的真心話。

## 核心方法論：深層需求路徑樹

### 這棵樹跟痛點樹有什麼不同？

痛點樹是**聚焦在一個產品上**，挖掘使用者對它的各種困擾。

深層需求路徑樹是**從需求出發**，展開所有可能的滿足方式。產品本身只是其中一條路徑。這讓你能看到：
- 競爭對手不只是同類產品，還包括完全不同的做法
- 使用者在不同路徑之間的心理拉扯
- 為什麼有些人最終選擇了看似不合理的方案

### 路徑樹的結構

```
深層需求（一句話描述）
├── 路徑一：使用這個產品
│   ├── 怎麼挑選
│   │   ├── 困難 → 心理反應 → 更深的想法
│   │   └── 岔路：替代挑法 → 也有問題
│   ├── 使用過程的問題
│   │   └── 困擾 → 試著解決 → 解法也有問題 → 內心OS
│   └── 使用之後
│       └── 感受 → 反思 → 掙扎
├── 路徑二：替代方案 A
│   ├── 優點 → 但是... → 更深的問題
│   └── 心理障礙 → 社會壓力 → 內心OS
├── 路徑三：替代方案 B
│   └── ...
├── 路徑四：最原始 / 最簡單的做法
│   └── 為什麼這反而可能最好 → 但也有缺點 → 掙扎
└── （可選）路徑五：乾脆不做了
    └── 放棄的心理過程
```

### 路徑樹的寫作原則

**1. 先找到深層需求，用一句話說清楚**

這是整棵樹的根。問自己：「如果這個產品消失了，使用者會用什麼方式來滿足同樣的渴望？」答案指向的那個渴望，就是深層需求。

深層需求要用使用者的語氣表達：
- ✓「想要舒舒服服的自慰」
- ✓「想要不費力地通勤」
- ✗「解決個人生理需求」（太學術）
- ✗「提升通勤效率」（太報告式）

**2. 路徑要涵蓋「完全不同類型的做法」**

不能只列同類產品。路徑之間的差異要夠大：
- 使用目標產品
- 花錢的替代方案（不同類型的產品或服務）
- 不花錢的替代方案
- 最原始的做法
- 社交性的解法（找人幫忙、建立關係）
- 放棄或接受現狀

路徑數量通常 3-5 條。不需要湊數，但要確保覆蓋了使用者真正會考慮的主要方向。

**3. 每條路徑都要有「往下鑽」的深度**

表面描述是骨架，深度才是價值。每條路徑至少要有：
- 2-3 個具體的子議題
- 每個子議題往下追 3-5 層
- 追到「心理掙扎」「社會壓力」「不好說出口的真心話」的層次

**4. 路徑之間要有「拉扯感」**

使用者不是冷靜地評估每條路徑然後選一個。他們是東想西想、猶猶豫豫、在不同路徑之間反覆掙扎。要呈現這種拉扯：
- 在路徑 A 的某個深處，突然冒出「不如直接去做路徑 B 算了」
- 在路徑 B 的某個困難處，回頭想「所以才需要路徑 A」
- 在路徑 C 的末端，感嘆「但其實路徑 D 就夠了」

**5. 內心 OS（斜體標示）是靈魂**

那些不好意思說出口的、社會不太接受的、不理性但超級真實的想法，用斜體標示。這些往往是最有洞察力的部分。每個主路徑下至少要有 3-4 個內心 OS。

範例：
- *花這麼多錢在自慰上，未免太空虛*
- *不如把錢花在興趣上，例如：電玩、旅行、美食*
- *其實只是想跟女生打情罵俏罷了*
- *內心希望現實生活很充實的人都消失*

**6. 因果鏈條，不是平行列舉**

每一層都要回答「然後呢？」「為什麼？」「那怎麼辦？」

壞的（平行列舉）：
```
- 去風俗店
  - 可能會染病
  - 很貴
  - 很麻煩
```

好的（因果鏈條）：
```
- 去風俗店
  - 可能會染病
  - 覺得和女生互動很麻煩
    - 所以才想在家裡自慰
  - 也未必會舒服
    - 到頭來還是自己最清楚自己舒服的點在哪
  - 風俗店裡的店員目光，讓人不舒服
    - 不知道店員對我的看法如何，讓人很難覺得舒服
  - 去風俗店的路上，會害怕遇到認識的人
    - 被問起要去哪裡，就一定要撒謊，內心很不舒服
    - 不知道他們會不會知道我要去風俗店
```

**7. 具體細節讓一切活起來**

搜尋到的真實數字、品牌名、使用場景都要融入。不是「價格偏高」，而是「一個要 3000 塊，買三個試試看就噴掉快一萬」。

## 執行步驟

### Step 1：上網搜尋（必須執行）

用 WebSearch 搜尋以下類型的資訊，至少進行 6-8 次搜尋：

1. **使用者對這個產品的真實討論**
   - `"[產品] 心得"` `"[產品] 推薦"` `"[產品] 評價"`
   - `"[產品] PTT"` `"[產品] Dcard"` `"[產品] 經驗分享"`

2. **使用者考慮過的替代方案**
   - `"[產品] 替代"` `"不用[產品] 怎麼辦"` `"[產品] vs"`
   - `"比[產品]更好的方法"` `"[深層需求] 方法"`

3. **使用者的心理描述**
   - `"[產品] 後悔"` `"[產品] 值得嗎"` `"[產品] 猶豫"`
   - `"[深層需求] 困擾"` `"[深層需求] 掙扎"`

4. **英文搜尋（擴大視野）**
   - `"[product] reddit"` `"[product] alternative"` `"[product] worth it"`
   - `"[deep need] how to"` `"[deep need] best way"`

搜尋的核心目的是：
- 找到使用者**真正嘗試過的替代方案**
- 找到他們**在不同方案之間切換的心理歷程**
- 找到那些**不好意思直說但很真實的想法**

### Step 2：用 WebFetch 深入閱讀

從搜尋結果中挑 3-4 個最有價值的頁面（論壇討論串、評論區），用 WebFetch 深入閱讀。重點抓取：
- 使用者描述自己「為什麼選擇 / 不選擇某個方案」的段落
- 使用者之間的辯論（A 說這個好，B 說那個好）
- 具體的數字、品牌名、使用場景
- 讓人會心一笑或心有戚戚焉的真實吐槽

### Step 3：辨識深層需求

這是最關鍵的一步。問自己：
- 如果這個產品不存在，使用者會怎麼做？
- 使用者買這個產品，真正想獲得的「狀態」是什麼？
- 用一句使用者會說的話來描述這個渴望

深層需求要能涵蓋所有替代路徑。如果你發現某條路徑跟深層需求對不上，要麼調整深層需求的描述，要麼那條路徑其實不該放進來。

### Step 4：建構路徑樹

切換到使用者視角，開始建構每條路徑的思考過程。

對每條路徑，想像一個真實的人在認真考慮這個做法：
- 他會先想到什麼？（表面考量）
- 然後呢？（深入一層）
- 遇到什麼困難？（現實障礙）
- 心裡怎麼想？（心理反應）
- 會不會想到其他路徑？（路徑間的拉扯）
- 有沒有不好意思說的？（內心 OS）

關鍵原則：
- **每條路徑都要深入到心理層面**，不能停在表面描述
- **融入搜尋素材**：把蒐集到的真實聲音編織進思考鏈條
- **加入你的推論**：人性的弱點、社會壓力、面子、懶惰、金錢考量——這些是思考的「肉」
- **語氣生活化**：像在 Dcard 或 PTT 上看到的發文語氣

### Step 5：產出 HTML 頁面

產出一個排版精美的靜態 HTML 頁面。CSS 和 HTML 全部放在同一個檔案裡。

#### 頁面結構

1. **頁首**：大標題 — `[產品名] 的深層需求探索`
2. **深層需求宣言**：一段醒目的引言區塊，用一句話點出深層需求，再用 2-3 句話說明「顧客如何嘗試滿足這個深層需求，以下是他們的可能做法和想法」
3. **路徑展開**：每條路徑用 H2 標題，底下是樹狀結構
4. **資料來源**：底部列出參考的網頁連結

#### 視覺設計規範

採用與痛點分析一致的暖色系研究筆記風格：

- **背景**：`linear-gradient(135deg, #f5f3f0 0%, #faf8f6 100%)`
- **主色調**：暖棕色系 `#d4a574` / `#c18a5a`
- **容器**：`max-width: 900px`，置中
- **字體**：`"Noto Sans TC", "Microsoft JhengHei", sans-serif`，行高 1.8
- **H1 標題**：漸層色文字（`background-clip: text`），大而醒目，置中
- **深層需求宣言區塊**：特殊的強調樣式——較大字體、置中、暖色背景框 `#fef5e7`，上下有裝飾線
- **H2 路徑標題**：左邊 5px 棕色色條 + 左 padding，格式為「一、路徑名稱」「二、路徑名稱」...
- **H3 子分類（可摺疊）**：
  - 每個 H3 區塊預設為**摺疊狀態**，只顯示 H3 標題
  - H3 標題左邊有一個展開/摺疊的箭頭指示器（`▶` 摺疊 / `▼` 展開）
  - 點擊 H3 標題即可展開/收合其下所有內容
  - 底部 2px 淡棕色線 `#e8d7c3`
  - 使用 `<details>` + `<summary>` HTML 原生標籤實作，不依賴 JavaScript
  - `<summary>` 的樣式要跟原本 H3 一致（字體大小、顏色、底線），cursor 設為 pointer
  - `details[open] > summary` 時箭頭從 `▶` 變成 `▼`
  - 展開/摺疊要有平滑感（summary 加上 hover 效果，例如背景微變色）
  - 具體 HTML 結構如下：
    ```html
    <details>
      <summary><h3>子分類標題</h3></summary>
      <div class="collapsible-content">
        <!-- 巢狀列表內容 -->
      </div>
    </details>
    ```
- **巢狀列表**（在 H3 摺疊區塊內）：
  - 自訂 bullet 符號 `▸`，棕色
  - 每層遞進縮排（25px → 20px → 18px）
  - 每層字體遞減（1em → 0.95em → 0.9em → 0.85em）
  - 每層顏色遞淡（`#2c2c2c` → `#4a4a4a` → `#666`）
- **內心 OS（`<em>` 標籤）**：
  - 粉色漸層背景 `linear-gradient(120deg, #fdeae8, #fce5df)`
  - 左邊 3px 棕色邊線
  - 特殊文字色 `#8b6f5a`
- **路徑間拉扯的提示**（例如「所以才想靠飛機杯解決」）：用稍不同的樣式讓讀者看出這是在路徑之間跳躍
- **資料來源區塊**：頂部棕色分隔線，菱形 bullet `◆`，連結棕色帶 hover 效果
- **Responsive**：768px 以下字體縮小

## 品質檢查清單

產出之前，逐項確認：

- [ ] 深層需求是否用一句使用者語氣的話清楚表達？
- [ ] 是否有 3-5 條本質不同的滿足路徑？（不能只是同類產品的比較）
- [ ] 每條路徑是否至少有 3-5 層深度？
- [ ] 路徑標題是否用使用者語氣？
- [ ] 是否有足夠的「內心 OS」斜體標示（每條路徑 3-4 個以上）？
- [ ] 路徑之間是否有「拉扯感」——在某條路徑中提到另一條路徑？
- [ ] 痛點之間是否有因果鏈條，而非平行列舉？
- [ ] 語氣是否像真人在自言自語，而非研究員在報告？
- [ ] 是否融入了搜尋到的真實素材（具體數字、品牌名、使用場景）？
- [ ] HTML 視覺風格是否為暖色系研究筆記風格？
- [ ] H3 區塊是否都用 `<details><summary>` 實作，預設摺疊？
- [ ] 摺疊的箭頭指示器和 hover 效果是否正常？
- [ ] 是否列出了資料來源？

## 檔案儲存

檔名格式：`深層需求探索_[產品名稱].html`

如果使用者的 CLAUDE.md 有指定輸出位置（例如 `產出 Outputs/`），就存到那裡。
